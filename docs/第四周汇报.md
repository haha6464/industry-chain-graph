# 第四周汇报：产业链图谱 Agent 进展与计划

汇报日期：2026-07-03

## 1. 本周进展

本周主要围绕 Agent 构建链路、前端工作流和图谱质量控制做优化。结合上次提交的 Agent/前端优化，目前系统已经从单次构建 demo，逐步调整为更适合多行业批量生产的分阶段流程。

### 1.1 Agent 构建流程拆分

原来的候选图谱构建和语义校验链路偏重，一次请求覆盖范围过大，耗时长且不利于定位问题。本周将流程拆成更清晰的阶段：

```text
搜索规划 -> 一级骨架构建与评估 -> 分支扩展与评估 -> 合并候选图谱 -> 最终硬规则校验 -> 必要时格式修复 -> 正式图谱
```

其中一级骨架和每个一级分支都会单独做质量评估；如果评估不通过，只针对当前骨架或当前分支请求修正，避免整图反复重跑。

### 1.2 校验职责重新划分

当前校验链路已经区分两类问题：

- 质量评估：关注产业链分类是否合理、分支是否均衡、节点粒度是否适合投研使用；
- 格式修复：只在硬规则校验失败时触发，重点修 JSON 结构、父子关系、level、关系类型等工程格式问题。

这样可以避免“语义质量评估”和“格式修复”混在一起，后续也方便把质量意见沉淀到复核流程里。

### 1.3 前端 Agent 工作流优化

前端已将 Agent 工作流拆成几个独立模块：

- 搜索规划；
- 一级骨架构建；
- 分支扩展与评估；
- 最终校验与格式修复；
- 增量更新；
- CSV 导出。

同时补充了运行日志、产物查看、产物删除能力。图谱展示页也优化了节点层级配色，改成连续色带，更容易看出层级深浅。

### 1.4 Prompt 口径调整

结合当前正式图谱的问题，本周重点调整了构建和评估 prompt。新的 prompt 更强调“证券/金融投研场景”的产业链口径，要求节点能够服务于行业比较、成本拆解、上下游传导和公司业务归因。

同时去掉了食品饮料行业特定表达，改成适用于 manifest 中多个行业的通用框架：

```text
上游供给 -> 生产/转换或服务交付 -> 产品/服务形态 -> 下游应用/需求 -> 基础设施与必要支撑
```

## 2. 当前问题

目前功能链路基本跑通，但图谱质量还需要继续打磨。

1. 多行业稳定性仍需验证  
   食品饮料 demo 能跑通，但 manifest 中还包括基础材料、工业、建筑、公用事业、交通运输、汽车等行业。不同类型行业的产业链结构差异较大，prompt 和评估规则还需要跨行业试跑验证。

2. 分支深度和节点粒度还需控制  
   当前图谱容易出现部分分支过深、部分核心分支过浅的问题；也可能把平台能力、解决方案、咨询服务等内容误抽成产业链节点。

3. 质量评分体系还不够完整  
   目前已有阶段性 LLM 评估和硬规则校验，但还缺少统一的质量分数，用于横向比较不同行业、不同版本的图谱质量。

## 3. 短期计划

1. 完善图谱评估模块  
   增加结构分数和 LLM 质量分数。结构分数关注节点规模、level 连续性、分支均衡、来源覆盖、关系完整性；LLM 分数关注投研分类质量、关键环节缺失、节点粒度和上下游逻辑。

2. 做多行业试点  
   优先选择 2-3 个差异较大的行业试跑，例如有色金属、汽车、公用事业或交通运输，观察一级骨架、分支扩展和质量评估是否稳定。

3. 优化人工复核体验  
   继续完善前端产物查看、删除、复核队列展示，后续可以加入节点改名、删除、合并、接受/拒绝建议等轻量编辑能力。

4. 清理旧链路和旧产物  
   已开始删除或替换旧的单体构建方式，后续继续清理不再使用的脚本、报告和前端入口，降低后续开发理解成本。

## 4. 后续方向

长期目标是把当前系统从“单行业 demo 构建工具”升级为“25 个基础行业图谱的半自动生产和维护系统”。

后续重点包括：

- 批量构建并评估 25 个行业图谱；
- 建立质量评分门槛，作为应用正式图谱前的判断依据；
- 沉淀人工复核意见，用于持续优化 prompt 和规则；
- 将增量更新、质量评估、人工确认串成闭环，支持图谱持续维护。

总体来看，本周的重点是把 Agent 链路从“大请求一次性生成”调整为“骨架和分支分阶段生成、逐步评估和修正”。下一步重点应放在多行业试点和评估指标固化上。
